## 基础知识

在开始之前，你需要了解几个核心概念：

### 什么是区块链追踪？

区块链是**公开透明**的账本。每一笔交易都记录在链上，任何人都可以查看资金从哪个地址流向了哪个地址。但地址本身是匿名的——你需要通过分析交易模式来推断地址背后的真实身份。

### 关键概念

| 概念 | 说明 |
|------|------|
| **钱包地址** | 类似银行账号，是一串字母数字组合（如 `bc1q...x7k9m`） |
| **UTXO** | 比特币的"余额"模型——不是账户余额，而是未花费交易输出的集合 |
| **KYC** | Know Your Customer，交易所的实名认证，是追踪的突破口 |
| **混币器** | 将多人资金混合再输出，打乱输入输出对应关系的服务 |
| **跨链桥** | 在不同区块链之间转移资产的桥梁（如 BTC → ETH） |

### 追踪的核心逻辑

```
暗网地址 ──→ 中间钱包（多层）──→ 出口（交易所/DeFi）
   ↑                                    ↑
 起点：已知非法              终点：有KYC或可关联真实身份
```

**核心思路：跟着钱走，找到出口。**

## 案例 #1: 直链追踪

### 📋 案件信息

- **难度**: ⭐
- **标题**: 直链追踪
- **简介**: 暗网市场"ShadowMart"被发现有一笔 0.8 BTC 的交易。请追踪这笔资金的最终去向。
- **目标**: 这笔资金最终流入了哪里？

### 🗺️ 交易网络

```
┌─────────────┐    0.8 BTC     ┌──────────┐    0.79 BTC    ┌──────────┐
│ ShadowMart  │ ─────────────→ │  钱包 A  │ ─────────────→ │  钱包 B  │
│  (暗网市场)  │   03-15 02:30  │  (未知)   │   03-15 04:15  │  (未知)   │
└─────────────┘                └──────────┘                └──────────┘
                                                                ↑
                                                            答案在这里
```

### 🔍 解题过程

**第一步：识别起点**

图中有一个红色节点「ShadowMart」，标记为暗网市场。这是资金的来源。

**第二步：追踪资金流向**

从 ShadowMart 出发，沿着箭头方向：
- ShadowMart → 钱包 A：0.8 BTC（03-15 02:30）
- 钱包 A → 钱包 B：0.79 BTC（03-15 04:15）

注意金额从 0.8 变成了 0.79——少了 0.01 BTC，这是**手续费**。这是判断资金链路连续性的重要线索。

**第三步：分析终点**

点击「钱包 B」查看详情，发现描述中提到：

> 未知钱包地址 1A2b...z8q3

这时候你需要判断：这个钱包的**真实身份**是什么？

**第四步：标记与提交**

- 把「钱包 B」标记为「🔵 交易所」
- 选中「钱包 B」，点击提交答案

**答案：钱包 B 实际上是交易所 B**

揭示后显示：
> 知名加密货币交易所，要求 KYC 实名认证，用户: zhang\*\*\*@email.com

### 📖 知识点

#### 1. 什么是 KYC？

KYC（Know Your Customer）是交易所的实名认证要求。用户需要提交身份证件才能交易，这使得交易所成为链上追踪的**关键突破口**。

在实际执法中，执法人员可以向交易所发出协查请求，获取 KYC 用户的真实身份信息。

#### 2. UTXO 模型

比特币不是"账户余额"模型，而是 **UTXO（未花费交易输出）** 模型。

- 每笔交易消耗之前的 UTXO 并产生新的 UTXO
- 形成一条可追溯的链路
- 类似现金：你有一张 100 元，买了 60 元的东西，收到 40 元找零——这 40 元就是一个新的 UTXO

## 案例 #2: 分叉迷踪

### 📋 案件信息

- **难度**: ⭐⭐
- **标题**: 分叉迷踪
- **简介**: 暗网论坛"DarkForum"的一笔赎金 2.5 BTC 被分散转入多个钱包。你需要追踪资金的聚合点。
- **目标**: 资金最终在哪个交易所被提现？

### 🗺️ 交易网络

```
                         ┌──────────┐    0.98    ┌──────────┐
                    ┌──→ │  钱包 A  │ ─────────→ │  钱包 D  │ ──┐
                    │    └──────────┘             └──────────┘   │
┌─────────────┐ 1.0 │                                            │    1.75   ┌──────────┐
│ DarkForum   │ ────┤    ┌──────────┐    0.78    ┌──────────┐   ├──────────→│  钱包 F  │
│  (暗网论坛)  │ 0.8 ├──→ │  钱包 B  │ ─────────→ │  钱包 D  │ ──┘           │  (答案)   │
│             │ ────┤    └──────────┘             └──────────┘               └──────────┘
│  赎金 2.5   │ 0.7 │                                                            ↑
└─────────────┘ ────┤    ┌──────────┐    0.68    ┌──────────┐    0.67           │
                    └──→ │  钱包 C  │ ─────────→ │  钱包 E  │ ─────────────────┘
                         └──────────┘             └──────────┘

时间线:
  10:00  DarkForum → A, B (分散)
  14:30  A, B → D (聚合)
  14:35  C → E (聚合)
  次日09:00  D, E → F (最终汇聚)
```

### 🔍 解题过程

**第一步：识别分散模式**

从 DarkForum 出发，资金被分成了 3 份：
- 1.0 BTC → 钱包 A
- 0.8 BTC → 钱包 B
- 0.7 BTC → 钱包 C

这是典型的**资金分散**手法——把一大笔钱拆成多份，通过不同路径转移，企图混淆追踪。

**第二步：追踪各条路径**

分别追踪三条路径：
- 路径 1: A → D（0.98 BTC，14:30）
- 路径 2: B → D（0.78 BTC，14:32）
- 路径 3: C → E（0.68 BTC，14:35）

**关键发现**：路径 1 和路径 2 的资金都流入了同一个节点「钱包 D」！

**第三步：时间关联分析**

注意时间线：
- A → D 和 B → D 的时间只差 **2 分钟**
- 这强烈暗示 A 和 B 的操作者是**同一个人**

**第四步：追踪聚合点**

- 钱包 D → 钱包 F：1.75 BTC（次日 09:00）
- 钱包 E → 钱包 F：0.67 BTC（次日 09:02）

三条路径最终都汇入了「钱包 F」——这就是答案。

**答案：钱包 F 是交易所 C**

> 加密货币交易所，KYC 用户: li\*\*\*@proton.me，IP: 103.42.xx.xx (北京)

### 📖 知识点

#### 1. 资金分散模式

犯罪分子常将大额资金分散到多个钱包，再通过不同路径聚合到交易所提现。这是一种常见的洗钱手法，目的是混淆追踪。

**识别特征**：
- 一笔大额资金在短时间内分成多笔
- 分散后的资金最终流入相同的少数几个地址

#### 2. 时间关联分析

即使资金经过多层分散，各路径的操作时间往往相近。通过分析时间关联，可以推断出哪些钱包属于同一控制人。

**实战技巧**：
- 分散操作通常在同一分钟或几分钟内完成
- 聚合操作也倾向于在相近时间进行
- 如果两条路径的操作时间高度一致，大概率是同一人控制

## 案例 #3: 混币迷雾

### 📋 案件信息

- **难度**: ⭐⭐⭐
- **标题**: 混币迷雾
- **简介**: 暗网市场"BlackBazaar"的一笔 1.2 BTC 交易经过了一个混币服务。混币器会打乱输入输出的对应关系，请想办法穿透迷雾。
- **目标**: 经过混币后，资金最终流向了哪里？

### 🗺️ 交易网络

```
┌──────────────┐   1.2    ┌──────────┐  1.19   ┌──────────┐  0.65   ┌──────────┐  0.64   ┌──────────┐
│ BlackBazaar  │ ───────→ │  钱包 A  │ ──────→ │  服务 X  │ ──────→ │  钱包 B  │ ──────→ │  钱包 D  │
│  (暗网市场)   │          │  (未知)   │         │  (未知)   │         │  (未知)   │         │  (答案)   │
└──────────────┘          └──────────┘         │          │         └──────────┘         └──────────┘
                                                │          │  0.53   ┌──────────┐  0.52        ↑
                                                │          │ ──────→ │  钱包 C  │ ──────→  交易所A │
                                                └──────────┘         └──────────┘
```

### 🔍 解题过程

**第一步：识别可疑节点**

从 BlackBazaar 出发追踪：
- BlackBazaar → 钱包 A：1.2 BTC
- 钱包 A → 服务 X：1.19 BTC

注意「服务 X」的描述：

> 未知服务，地址: bc1q...mix01，大量资金流入流出

**大量资金流入流出**——这是混币器的典型特征！

**第二步：理解混币器**

点击「服务 X」查看详情。虽然此时还不知道它的真实身份，但你可以推断：

- 输入：1.19 BTC（来自钱包 A）
- 输出：0.65 + 0.53 = 1.18 BTC

输入和输出金额几乎相等（差 0.01 是手续费），而且**一笔进、两笔出**——这是混币器将资金拆分输出的表现。

**第三步：穿透混币器**

混币器的目的是打乱输入输出的对应关系。但我们可以用以下方法穿透：

1. **时间关联**：混币器的输出时间（05:20、05:22）与输入时间（03:45）相差约 1.5 小时，这是典型的混币延迟
2. **金额守恒**：输入 1.19 ≈ 输出 0.65 + 0.53 = 1.18，金额基本守恒
3. **后续聚合**：钱包 B 和钱包 C 的资金最终都流入了同一个地址

**第四步：追踪终点**

- 钱包 B → 钱包 D：0.64 BTC
- 钱包 C → 钱包 D：0.52 BTC

两条路径在「钱包 D」聚合——这是交易所 A。

**答案：钱包 D 是交易所 A**

> 大型加密货币交易所，KYC 用户: wang\*\*\*@gmail.com，已通报执法部门

### 📖 知识点

#### 1. 混币器原理

混币器（Mixer/Tumbler）将多人的资金混在一起，再分别输出等额资金。外部观察者无法直接将输入和输出对应起来。

```
用户A: 1 BTC ──┐
               ├──→ [混币器] ──→ 输出 X: 0.8 BTC (可能是A的，也可能是B的)
用户B: 1 BTC ──┘                ──→ 输出 Y: 1.2 BTC (同上)
```

#### 2. 穿透混币的方法

| 方法 | 说明 |
|------|------|
| **时间关联** | 输入后不久的输出最可能对应。混币器通常有固定延迟 |
| **金额分析** | 扣除手续费后的金额有规律。如果输入 1.2，输出中找 1.19 附近的 |
| **行为模式** | 同一人控制的地址有相似的交易习惯（如固定时间段操作） |
| **聚类分析** | 如果混币后的多个输出最终流入同一地址，它们很可能属于同一人 |

## 案例 #4: 跨链迷局

### 📋 案件信息

- **难度**: ⭐⭐⭐⭐
- **标题**: 跨链迷局
- **简介**: 一个黑客团伙将 3.2 BTC 赃款通过跨链桥转换为 ETH，试图逃避比特币链上的追踪。你需要跨越不同区块链追踪资金。
- **目标**: 资金最终在哪个平台被兑换为法币？

### 🗺️ 交易网络

```
                         [比特币链]                                    [以太坊链]
                         ┌──────────┐   1.98   ┌──────────┐  28.5 ETH ┌──────────┐  28.3 ETH ┌──────────┐
                    ┌──→ │ BTC 钱包A │ ───────→ │          │ ────────→ │ ETH钱包A │ ────────→ │          │
┌──────────┐  2.0 ─┤    └──────────┘          │  跨链桥   │          └──────────┘          │  DeFi    │
│ 黑客钱包  │ ─────┤                           │  服务    │                                  │  平台    │
│ (赃款)    │  1.2 ┤    ┌──────────┐   1.18   │          │  16.8 ETH ┌──────────┐  16.7 ETH │  (答案)  │
└──────────┘ ──────┴──→ │ BTC 钱包B │ ───────→ │          │ ────────→ │ ETH钱包B │ ────────→ │          │
                         └──────────┘          └──────────┘          └──────────┘          └──────────┘

链上变化:  BTC ──跨链桥──→ ETH ──DEX──→ USDC ──交易所──→ 法币
```

### 🔍 解题过程

**第一步：识别跨链转移**

从黑客钱包出发：
- 黑客钱包 → BTC 钱包 A：2.0 BTC
- 黑客钱包 → BTC 钱包 B：1.2 BTC

然后：
- BTC 钱包 A → 跨链服务：1.98 BTC
- BTC 钱包 B → 跨链服务：1.18 BTC

**关键发现**：BTC 链上的资金流入了一个「跨链服务」，但之后出现的节点变成了 **ETH 钱包**——币种变了！

**第二步：理解跨链桥**

跨链桥是一种在不同区块链之间转移资产的服务：
- 用户在链 A 上存入资产
- 跨链桥在链 B 上铸造等值资产
- 原链上的资产被锁定或销毁

犯罪分子利用跨链桥来**切断同一条链上的追踪链路**。

**第三步：跨链关联**

虽然币种变了，但可以通过以下方式关联：
- **时间关联**：跨链桥的输入和输出时间通常在几分钟到几小时内
- **金额关联**：BTC 按汇率转换为 ETH，金额有对应关系
  - BTC 钱包 A 存入 1.98 BTC → ETH 钱包 A 收到 28.5 ETH（约 1:14.4 的汇率）
  - BTC 钱包 B 存入 1.18 BTC → ETH 钱包 B 收到 16.8 ETH（汇率一致）

**第四步：追踪以太坊链**

- ETH 钱包 A → DeFi 平台：28.3 ETH
- ETH 钱包 B → DeFi 平台：16.7 ETH

**答案：DeFi 平台（DEX 聚合器）**

> 去中心化交易所聚合器，通过流动性池兑换为 USDC，关联 KYC 交易所出金记录

### 📖 知识点

#### 1. 跨链桥洗钱

犯罪分子利用跨链桥将资金从一条区块链转移到另一条，试图切断追踪链路。

**为什么有效**：
- 比特币链上的追踪工具看不到以太坊链上的交易
- 不同链使用不同的地址格式，无法直接对应

**如何突破**：
- 跨链桥的两端交易可通过时间和金额进行关联
- 跨链桥本身可能保留日志（中心化跨链桥）
- 跨链桥的智能合约交易是公开的

#### 2. DeFi 监管

虽然 DeFi 平台本身不要求 KYC，但最终兑换为法币时通常需要通过中心化交易所——这就是追踪的突破口。

**资金流向**：
```
赃款 → 跨链桥 → DeFi兑换 → 中心化交易所(有KYC) → 法币
                                    ↑
                              追踪的关键节点
```

## 案例 #5: 终极追踪

### 📋 案件信息

- **难度**: ⭐⭐⭐⭐⭐
- **标题**: 终极追踪
- **简介**: 一个高级持续威胁（APT）组织通过钓鱼攻击窃取了 5.8 BTC。他们使用了混币器、跨链桥、多层钱包等所有手段。这是最复杂的追踪任务。
- **目标**: 资金最终从哪个平台出金为法币？

### 🗺️ 交易网络

```
[比特币链]                                                    [以太坊链]
┌──────────┐  3.5  ┌────────┐         ┌────────┐  3.18  ┌──────────┐  45.2 ETH ┌──────────┐  45.0 ETH ┌──────────┐
│ 钓鱼攻击  │ ───→ │ 钱包 A │ ──┐     │ 钱包 C │ ─────→ │  转换    │ ────────→ │ ETH钱包  │ ────────→ │ 出金平台  │
│ (入口)    │      └────────┘   │ ┌──→│        │        │  服务    │          │          │          │  (答案)  │
│          │  2.3  ┌────────┐   │ │   └────────┘        │          │          │          │          │          │
│          │ ───→ │ 钱包 B │ ──┤ │                      └──────────┘          └──────────┘          └──────────┘
└──────────┘      └────────┘   │ │                           ↑
                           ┌───┘ │   3.2                2.5
                           │     │   ┌────────┐         ┌────────┐
                           │     └──→│ 服务 M │────────→│ 钱包 D │───┘
                           │         │ (混币器) │         └────────┘
                           │         └────────┘
                           └───────────────────────────────┘

完整链路:
  钓鱼攻击 → BTC分散 → 混币器 → BTC聚合 → 跨链桥 → ETH → 出金平台
```

### 🔍 解题过程

**第一步：识别攻击入口**

图的最左侧是「钓鱼攻击」节点，描述为：

> APT组织的钓鱼入口，伪造交易所登录页

这是资金的来源——犯罪分子通过钓鱼窃取了受害者的加密货币。

**第二步：追踪分散**

- 钓鱼攻击 → 钱包 A：3.5 BTC
- 钓鱼攻击 → 钱包 B：2.3 BTC

资金被分成两路。这是第一层混淆。

**第三步：发现混币器**

- 钱包 A → 服务 M：3.48 BTC
- 钱包 B → 服务 M：2.28 BTC

两路资金都流入了「服务 M」——描述为「可疑服务，大量资金中转」。这是混币器。

**第四步：穿透混币**

混币器输出：
- 服务 M → 钱包 C：3.2 BTC（09-02 03:00）
- 服务 M → 钱包 D：2.5 BTC（09-02 03:02）

输出时间只差 2 分钟，且金额之和（3.2 + 2.5 = 5.7）接近输入之和（3.48 + 2.28 = 5.76）。这确认了它们来自同一组输入。

**第五步：跨链转移**

- 钱包 C → 转换服务：3.18 BTC
- 钱包 D → 转换服务：2.48 BTC

两路资金汇入「转换服务」——这是跨链桥。之后资金出现在以太坊链上：

- 转换服务 → ETH 钱包：45.2 ETH

**第六步：找到出口**

- ETH 钱包 → 出金平台：45.0 ETH

「出金平台」的描述揭示了最终身份：

> 加密货币场外交易平台
> KYC: chen\*\*\*@icloud.com
> 银行账户: 工商银行 \*\*\*\*3847
> 已冻结，执法部门介入

**答案：出金平台（场外交易所）**

### 📖 知识点

#### 1. APT 攻击链

高级持续威胁组织通常采用多层洗钱：

```
分散 → 混币 → 跨链 → 聚合 → 出金
```

每一层都增加追踪难度。但每一步都会留下痕迹——关键是**耐心**和**系统性**地分析。

#### 2. 链上分析工具

专业工具如 Chainalysis、Elliptic 可以：
- 自动标记已知的混币器、交易所地址
- 通过聚类算法关联同一控制人的钱包
- 可视化交易网络图
- 设置可疑交易警报

#### 3. 执法协作

跨国加密货币犯罪需要多国执法协作：
- 交易所的 KYC 数据是锁定嫌疑人身份的最终突破口
- 银行账户冻结需要当地执法部门配合
- 国际刑警组织（Interpol）协调跨国案件

## 通用技巧总结

### 🔑 追踪五步法

```
1. 识别起点 → 找到资金来源（暗网/黑客钱包）
2. 顺藤摸瓜 → 沿着箭头方向追踪每一笔交易
3. 识别手法 → 判断使用了什么混淆手段（分散/混币/跨链）
4. 穿透迷雾 → 用时间/金额/行为分析突破混淆
5. 锁定出口 → 找到最终可以关联真实身份的节点
```

### 🎯 关键突破口

| 突破口 | 说明 |
|--------|------|
| **交易所 KYC** | 最直接的身份关联，执法协查的首选 |
| **时间关联** | 即使经过混币，输入输出的时间差也能暴露关联 |
| **金额守恒** | 扣除手续费后，输入输出金额基本守恒 |
| **聚类分析** | 多个地址最终流入同一地址 → 可能属于同一人 |
| **行为模式** | 操作时间、交易习惯、Gas 设置等都能暴露身份 |

### ⚠️ 常见陷阱

1. **不要只看一层** — 犯罪分子通常使用多层混淆
2. **不要忽略小额** — 有时小额交易是测试，大额紧跟其后
3. **不要只看一条链** — 跨链桥会让资金消失在另一条链上
4. **注意手续费** — 手续费的变化可以判断是否经过了中间服务

### 📊 得分优化

- **少用提示** — 每个提示扣 50 分
- **快速作答** — 时间越短，时间奖励越高
- **正确标记** — 每个正确标记 +100 分
- **一次性答对** — 答错没有惩罚但浪费时间
